### Jingjing One Page V0 修改策略

#### 近期任务 (可快速迭代)

这些是可以在短期内快速完成的修改，主要涉及现有组件的调整和内容更新。

- [x] **移除 `Cross Selling` 部分:**
    *   **描述:** 直接从页面布局中移除 `Cross Selling` 相关的组件和数据。
    *   **实现:** 在 `JingjingOnePage_V0/index.tsx` 中定位并删除该模块。同时清理 `data.ts` 中不再需要的数据。
    *   **UI 依赖:** 无，属于页面级修改。

- [x] **移除 `Peak Hours` 和 `Category Mix`:**
    *   **描述:** 从页面中移除这两个图表/模块。
    *   **实现:** 在 `JingjingOnePage_V0/index.tsx` 中定位并删除。同时清理 `data.ts` 中相关的数据源。
    *   **UI 依赖:** 无，属于页面级修改。

- [x] **增加 AI 交互按钮:**
    *   **描述:** 在页面右上角或合适位置增加一个机器人图标按钮，为未来的 AI 功能预留入口。
    *   **实现:** 使用 `components/ui/IconButton` 组件创建一个新的按钮实例。
    *   **UI 依赖:** 依赖现有的 `@ui/IconButton` 组件。如果现有图标不满足需求，可能需要向 `Icon` 组件库中新增一个机器人图标。

- [x] **`Hot Seller` 部分增加 `XStore` 和 `Omni` 切换:**
    *   **描述:** 在 `Hot Seller` 模块标题旁或内容区顶部增加一个切换开关，用于在 `XStore` 和 `Omni` 视图间切换。
    *   **实现:** 使用 `components/ui/Switch` 或一组 `Button` 组件来实现切换逻辑。需要更新 `data.ts` 以提供两种模式的数据或在前端实现数据过滤。
    *   **UI 依赖:** 依赖现有的 `@ui/Switch` 或 `@ui/Button` 组件。

- [x] **拆分 `Product Opportunity` 部分:**
    *   **描述:** 将现有的 `Product Opportunity` 模块重构为四个新的子模块。
    *   **实现:** 修改 `JingjingOnePage_V0/index.tsx` 中的布局，创建四个新的内容区块，并从 `data.ts` 中获取对应的数据进行填充。
    *   **UI 依赖:** 可能需要使用 `@ui/Card` 或其他容器组件来组织这四个部分，使其视觉上清晰地区分开。

#### 中期任务

这些任务需要更深入的开发或新组件的创建。

- [x] **增加 `Weekly` 节奏图表:**
    *   **描述:** 创建一个类似瀑布图的周视图 (Mon-Sun)，以展示一周内的数据变化趋势。
    *   **实现:**
        *   **方案 A (推荐):** 在 `components/ui/forDashboard/` 目录下创建一个新的、可复用的图表组件，例如 `WaterfallChart`。这能确保组件的通用性和未来的可维护性。
        *   **方案 B (快速但不推荐):** 在 `JingjingOnePage_V0` 页面内部临时实现这个图表。这种方式不利于代码复用。
    *   **UI 依赖:** 需要新建一个 `@ui/forDashboard/WaterfallChart` 组件。这涉及到图表库(如 `recharts` 或 `d3`)的使用和封装。

#### 长期任务

这些是关于未来功能的规划。

- [ ] **实现 AI 交互功能:**
    *   **描述:** 当前只是增加了一个入口按钮。未来的工作是定义点击按钮后的具体交互逻辑，例如弹出对话框、显示智能分析结果等。
    *   **实现:** 这将是一个全新的功能模块，需要产品、设计和开发的共同投入。
    *   **UI 依赖:** 需要设计和开发新的 UI 组件，如对话框 (Modal/Dialog), 数据展示面板等。
